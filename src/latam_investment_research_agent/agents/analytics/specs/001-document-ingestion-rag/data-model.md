# Data Model: Document Ingestion & RAG Query Agent Framework

**Branch**: `001-document-ingestion-rag` | **Date**: 2026-05-23

## LangGraph State Schemas

### IngestionState (Internal — Ingestion Graph)

The mutable state passed between nodes of the ingestion LangGraph.

| Field | Type | Description |
|---|---|---|
| `source_reference` | `str` | URL or absolute file path of the source document |
| `raw_content` | `str \| None` | Raw text extracted from the document (pages joined) |
| `extracted_datasets` | `list[ExtractedDataset]` | All datasets found by the extraction node |
| `current_dataset_index` | `int` | Index into `extracted_datasets` for the routing loop |
| `ingestion_results` | `list[DatasetIngestionResult]` | Accumulates successful write results |
| `ingestion_failures` | `list[DatasetIngestionFailure]` | Accumulates failed write details |
| `error` | `str \| None` | Set if a fatal error (e.g., document fetch failure) halts the graph |

---

### IngestionGraphInput (Public — Caller-facing)

The input supplied by the parent orchestrator when invoking the ingestion graph.

| Field | Type | Default | Description |
|---|---|---|---|
| `source_reference` | `str` | required | URL (PDF or web page) or local file path |
| `export_row_limit_override` | `int \| None` | `None` | Not used by ingestion; reserved for future |

---

### IngestionSummary (Public — Returned to Caller)

The structured output returned by the ingestion graph upon completion.

| Field | Type | Description |
|---|---|---|
| `source_reference` | `str` | The source document that was processed |
| `total_datasets_found` | `int` | Number of datasets the extraction node identified |
| `datasets_succeeded` | `list[DatasetIngestionResult]` | Datasets successfully written |
| `datasets_failed` | `list[DatasetIngestionFailure]` | Datasets that could not be written |

---

### ExtractedDataset

Represents one semantically coherent numerical dataset extracted from a source document section.

| Field | Type | Description |
|---|---|---|
| `dataset_name` | `str` | Human-readable name inferred by the LLM (e.g., "Annual Coffee Production by Region") |
| `context_labels` | `list[str]` | Surrounding text labels that describe the data's meaning |
| `column_names` | `list[str]` | Column headers extracted from the table |
| `rows` | `list[dict[str, Any]]` | Rows as key-value maps (column_name → value) |
| `content_hash` | `str` | Full SHA-256 hex digest of `json.dumps(row, sort_keys=True, default=str)` — computed per-row at write time, not per-dataset |

---

### RoutingDecision

The structured output of the LLM-based table routing node for one dataset.

| Field | Type | Description |
|---|---|---|
| `target_table_name` | `str` | Existing table name or `"__create_new__"` sentinel |
| `routing_action` | `Literal["append", "create"]` | Whether to append or create |
| `rationale` | `str` | LLM explanation of the routing decision (logged, not returned to caller) |
| `proposed_schema` | `list[ColumnDefinition] \| None` | Column definitions for new table creation; `None` for append |

---

### ColumnDefinition

Describes one column in a proposed new ClickHouse table schema.

| Field | Type | Description |
|---|---|---|
| `column_name` | `str` | Snake_case column name |
| `clickhouse_type` | `str` | ClickHouse type string — monetary amounts MUST use `Decimal(18,4)`, not `Float64`; use `Float64` only for non-monetary ratios/percentages |
| `description` | `str` | What this column represents |

---

### DatasetIngestionResult

Records the outcome of a successful dataset write.

| Field | Type | Description |
|---|---|---|
| `dataset_name` | `str` | Name of the extracted dataset |
| `target_table_name` | `str` | ClickHouse table where rows were written |
| `routing_action` | `Literal["append", "create"]` | Whether the table was appended or created |
| `rows_written` | `int` | Number of rows inserted (excluding deduplicated duplicates) |

---

### DatasetIngestionFailure

Records the outcome of a failed dataset write attempt.

| Field | Type | Description |
|---|---|---|
| `dataset_name` | `str` | Name of the extracted dataset |
| `error_detail` | `str` | Exception message or error description |

---

### RAGQueryState (Internal — RAG Graph)

The mutable state passed between nodes of the RAG query LangGraph.

| Field | Type | Description |
|---|---|---|
| `natural_language_question` | `str` | The question received from the parent orchestrator |
| `export_row_limit` | `int` | Maximum rows to include in the CSV export (default 10,000) |
| `available_table_schemas` | `list[TableSchema]` | All tables + columns retrieved from ClickHouse |
| `selected_table_names` | `list[str]` | Table names the LLM selected as relevant |
| `assembled_sql_queries` | `list[str]` | SQL queries generated by the LLM |
| `query_result_records` | `list[dict[str, Any]]` | Rows returned by executing all queries |
| `export_file_path` | `str \| None` | Absolute path to the written CSV file |
| `rationale` | `str \| None` | LLM explanation of why the data answers the question |
| `error` | `str \| None` | Set if a fatal error halts the graph |

---

### RAGQueryInput (Public — Caller-facing)

| Field | Type | Default | Description |
|---|---|---|---|
| `natural_language_question` | `str` | required | The question to answer from the database |
| `export_row_limit` | `int` | `10000` | Maximum rows in the CSV export |
| `export_directory` | `str` | `"./exports"` | Directory where CSV files are written |

---

### RAGQueryOutput (Public — Returned to Caller)

| Field | Type | Description |
|---|---|---|
| `export_file_path` | `str` | Absolute path to the exported CSV file |
| `rationale` | `str` | Human-readable explanation of why this data answers the question |
| `sql_queries_used` | `list[str]` | The exact SQL queries executed |
| `row_count` | `int` | Number of rows in the CSV (may be less than limit if fewer results exist) |
| `was_truncated` | `bool` | `True` if results were truncated to `export_row_limit` |

---

### TableSchema

Represents the schema of one ClickHouse table, used by both the routing and RAG nodes.

| Field | Type | Description |
|---|---|---|
| `table_name` | `str` | ClickHouse table name |
| `columns` | `list[ColumnInfo]` | All columns in the table |

---

### ColumnInfo

| Field | Type | Description |
|---|---|---|
| `column_name` | `str` | Column name |
| `column_type` | `str` | ClickHouse type string |

## ClickHouse Table Conventions

Every dynamically created table MUST include these mandatory audit columns:

| Column | Type | Description |
|---|---|---|
| `source_reference` | `String` | URL or file path of the source document |
| `ingestion_timestamp` | `DateTime64(3, 'UTC')` | UTC time of row insertion |
| `content_hash` | `String` | SHA-256-derived row fingerprint for deduplication |

**Engine**: `MergeTree()`
**Primary key**: `(source_reference, content_hash)`
**Order by**: `(source_reference, content_hash, ingestion_timestamp)`

## Entity Relationships

```text
IngestionGraphInput
  └── drives → IngestionState
        ├── source_reference: str
        ├── extracted_datasets: list[ExtractedDataset]
        │     └── each → RoutingDecision
        │                 └── ColumnDefinition[] (for new tables only)
        ├── ingestion_results: list[DatasetIngestionResult]
        └── ingestion_failures: list[DatasetIngestionFailure]
  └── produces → IngestionSummary

RAGQueryInput
  └── drives → RAGQueryState
        ├── available_table_schemas: list[TableSchema]
        │     └── ColumnInfo[]
        ├── assembled_sql_queries: list[str]
        └── query_result_records: list[dict]
  └── produces → RAGQueryOutput
```

## State Lifecycle

### Ingestion Graph

```
IngestionGraphInput → [fetch_document_node] → raw_content populated
  → [extract_numerical_data_node] → extracted_datasets populated
  → [route_dataset_node]* → routing decision per dataset (* runs once per dataset)
  → [write_to_clickhouse_node]* → result or failure appended
  → [build_ingestion_summary_node] → IngestionSummary returned
```

### RAG Query Graph

```
RAGQueryInput → [introspect_schema_node] → available_table_schemas populated
  → [select_relevant_tables_node] → selected_table_names populated
  → [assemble_queries_node] → assembled_sql_queries populated
  → [execute_queries_node] → query_result_records populated
  → [export_results_node] → export_file_path populated
  → [build_rag_response_node] → RAGQueryOutput returned
```
